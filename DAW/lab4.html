Definiciones iniciales:
  1. URL(Uniform Resource Locator): Es una manera de encontrar una página en la web. El url se refiere al final a algún archivo ubicado en una máquina de alguna red, siendo esta máquina el servidor de una dirección. Un URL contiene uno o más de los siguientes elementos: Nombre del host, nombre de archivo, número de puerto, referencia etc.
  2. MIME (MUltipurpose Internet Mail Extentions): Es un estándar que indica el fomato de algún archivo o documento. Por ejemplo texto plano, html, javascript, css, png, mp4, etc.
  3. Existen una serie de códigos de error, son parte del protocolo HTTP, dichos códigos son usualmente: 404 que significa que no se encontró un elemento y cualquier código 5xx; ya que, estos significan alguna falla del servidor que aloja a la página.
    4. XML (eXtensible Markup Language): Fue diseñado para guardar datos y poderlos transportar. Una de las características importantes en el diseño del XML es que fue hecho con el fin de poder ser interpretado por humanos, como también por computadoras.
  4. XHTML (EXtensible HyperText Markup Language): Es un lenguaje muy parecido al HTML; sin embargo, es diferente. Fue desarrollado para hacer a HTML más extenso y añadir operabilidad con más formas de datos. La manera de manejar errores para XHTML es mucho más estricta que aquella de HTML.
  5. HTML5: Es la tecnología más actualizada del estándar HTML. Significa Hypertext Markup Language. En si fue diseñado para poder ser utilizado por cualquier navegador web abierto, como también con el fin de mejorar la semántica de HTML4, conectividad, para almacenar datos de manera local, para mejorar algunas características de multimedia, mejora de rendimiento, entre muchas otras actualizaciones importantes.
  6. JS (JavaScript): Es un lenguaje de programación ligero interpretado. Se utiliza para programar scripts en las páginas web usualmente. También es utilizado a nivel servidor y en muchas otras aplicaciones.
  7. PHP (Hypertext Preprocessor): Un lenguaje de propósito general de fuente abierta, que puede ser embedido en HTML, por lo que es de mucha utilidad para el desarrollo web.
  8. AJAX (Asynchronous JavaScript And XML): Un set de técnicas de desarrollo web que se utiliza para crear apolicacionies asíncronas web. Esto quiere decir que pueden enviar o recibir peticiones de manera asíncrona de un servidor.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Primero que nada: HTTP define un conjunto de métodos de petición para indicar la acción que se desea realizar para un recurso determinado. Aunque estos también pueden ser sustantivos, estos métodos de solicitud a veces son llamados HTTP verbs. Cada uno de ellos implementan una semántica diferente, pero algunas características similares son compartidas por un grupo de ellos: ej. un request method puede ser safe, idempotent, o cacheable.[1]


  • GET: Solicita un recurso. Esta petición sólamente recupera datos.
  • HEAD: Pide lo mismo que el método GET; sin embargo, no pide el cuerpo de la respuesta.
  • POST: Este método se utiliza para enviar a una entidad a un recuros específico.
  • PUT: Crea un nuevo elemento o lo reemplaza con los datos de la petición.
  • PATCH: Utilizado para aplicar modificaciones parciales a un recurso.
  • DELETE: Borra un elemento en específico.
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debería de utilizar el método POST; ya que, este método pide una respuesta en base a datos que se envían desde un navegador.

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Un "!important" debería ser utilizado en casos muy específicos. Especialmente cuando tenemos algún selector global de CSS y queremos para un elemento en específico de ese selector algo diferente. Otra opción sería cuando se aplica un estilo directamente en el tag de HTML y queremos sobreescribirlo de alguna manera. Recordemos que aplicar estilos directos no es buena práctica.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Debido a que una imagen con tamaño incorrecto se vería deformada y en consecuencia sería poco estético.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?Dependiendo de los dispositivos a los que va orientado un sitio, es buena idea considerar los posibles tipos de pantalla que se van a utilizar, considerando los pixeles por pulgada. Esto se debe a que las medidas pt y px tienen las siguientes características:
  • 1 pt = 1/72 de pulgada
  • 1 px = 1/96 de pulgada
¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?Debido a que evita cualquier caracter extra en la página como por ejemplo: Espacios, tabulares, etc. Al final no hace una gran diferencia, pero si ayuda de alguna manera. Esto impacta aún más cuando se habla de CSS más complejos o pesados.

¿Qué métodos tiene el objeto Date? (Menciona al menos 5*)Algunos de ellos son:
  • getFullYear(): Obtiene el año actual en un dígito de cuatro números. Ejemplo: 2019.
  • getMonth(): Obtiene el mes actual en un número del 0 al 11. Ejemplo: Enero : 0, Julio: 6
  • getDate(): Obtiene el día como un número del 1 al 31.
  • getHours(): Obtiene la hora actual del 0 al 23. Ejemplo: Si son las 12 AM: 00
